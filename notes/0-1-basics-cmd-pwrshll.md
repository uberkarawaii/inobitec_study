**Командные оболочки Windows: PowerShell и cmd.** Современная основная оболочка разработчика на Windows — 
**PowerShell** (cmdlet'ы по схеме `Verb-Noun`, объектная модель пайплайна вместо текстового, 
полноценный скриптовый язык, кросс-платформенность через PowerShell 7). Классический `cmd` остаётся для 
совместимости со старыми сценариями `.bat` и инструментами, которые от него зависят.
Понимание разницы между объектным пайплайном (передаёт типизированные объекты) 
и текстовым пайплайном (передаёт строки), профили (`$PROFILE`), execution policy.
-------------------Power Shell-----------------------------
*терминал - это именно UI который показывает работу оболочки*

power shell:

- оболочка командой строки

- возможность писать исполняемые инструкции

- платформа для управления конфигурацией
  конфигурация - это состояние компа. что установлено, какие права у пользователя, и т. д.
  администратор не будет руками настраивать все компы. 
  легче написать один скрипт и выполнить его для всех компов, тогда они будут в нужном состоянии
  платформа позволяет сделать описанное выше, а также следить и поддерживать состояние платформы

Power Shell принимает и отдаёт объекты .NET, в не текст. 
и когда делается цепочка команд через | то результат операции передаётся в след. 
операцию не как текст, а как объект .NET

в Power Shell тоже можно делать cd, mkdir, ... Linux-style команды
но они - лишь псевдонимы для реальных команд Power Shell с таким же значением; для проверки псевдонимов: Get-Alias ...
стиль команд Power Shell: Verb-Noun. пишется в CamelCase хотя сам PowerShell не чувствителен к регистру

т.к. работа тут именно с объектами, то можно сделать не только:
``` 
PS C:\Users\User> Get-ChildItem | Select-Object -Last 1 | Select-Object -Property Name

Name
----
urllib3-1.26.7-py2.py3-none-any.whl
```

но и так:
```
PS C:\Users\User> (Get-ChildItem | Select-Object -Last 1).Name
urllib3-1.26.7-py2.py3-none-any.whl
```

в первом случае это отфильтрованный объект, у которого оставлено 1 свойство
во втором случае выводится значение поля 
т.е. при выводе "табличек" Power Shell показывает именно объекты
